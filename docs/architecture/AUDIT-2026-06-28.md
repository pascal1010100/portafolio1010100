# Auditoría arquitectónica inicial — 2026-06-28

## Resumen

La visión de Pascal.dev es más madura que su baseline técnico. La prioridad es reproducibilidad, contratos, evidencia y límites claros antes de incorporar automatización avanzada.

## Fortalezas

- Dirección de producto y visual explícita.
- TypeScript estricto.
- Validación y sanitización básica del formulario.
- 3D aislado, diferido y compatible con movimiento reducido.
- App Router, metadata y páginas estáticas de proyectos.

## Riesgos priorizados

1. Ausencia de pruebas automatizadas.
2. Rate limit en memoria para un entorno distribuido.
3. Contenido de proyectos sin esquema de evidencia ni procedencia.
4. Exceso de límites de cliente y dependencias/componentes heredados.
5. Imágenes sin optimización y activos pesados.
6. Inconsistencia entre páginas nuevas y superficies heredadas.

## Deuda registrada para fases posteriores

- Unificar hooks y toasts duplicados.
- Inventariar y retirar dependencias/componentes sin uso.
- Crear esquema validado para proyectos y servicios.
- Reducir client boundaries.
- Implementar pruebas de API, navegación, responsive y accesibilidad.
- Sustituir rate limiting local por infraestructura durable.
- Añadir procedencia, verificación y aprobación al contenido.

No se crearán agentes funcionales, monorepo, CMS, cola ni base de datos durante Fase 0.
